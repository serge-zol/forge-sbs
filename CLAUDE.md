# Кузня. Крок за кроком. · forge-sbs

## Суть проекту
Кузня — фабрика AI-агентів. Тут створюються, перевіряються, версіонуються і зберігаються інструкції (system prompts). Ціль — бібліотека перевірених інструкцій із доказовою якістю (числові метрики, не декларації), розгортуваних на будь-якій платформі.

**Конкурентність живе не в GENERATE, а в TEST + накопиченні.** Фабрика розумнішає з часом через golden set і playbook.

Серж керує через Dispatch з телефона (19:00–23:00). Наві виконує технічну роботу.

## Воркфлоу

INBOX(карантин) → INPUT → GENERATE → REVIEW → TEST → STORE → [PROD → flywheel]
                                 ^          |       |
                                 +-- FAIL --+-------+

0. INBOX КАРАНТИН — inbox/ = недовірена зона. Санітизація до GENERATE.
1. INPUT — Серж описує агента: що робить, для кого, контекст
2. GENERATE — Коваль як пайплайн: spec → каркас → секції → самокритика → стиск у core. Best-of-3.
3. REVIEW — ЧЕК (GPT-5.5, напівручно), рубрика 1-5. Поріг 4.0. FAIL → GENERATE з фідбеком
4. TEST — Pairwise + Beat-baseline gate: нова версія мусить бити голу базову модель.
5. STORE — agents/[name]/vN/ + git commit + registry.md. Reproducibility ledger: пін версії + лог per vN.
6. PROD → flywheel — провал в бою → новий golden-case. Бібліотека вчиться з продакшену.

Перший пріоритет: inbox-карантин → рубрика + pairwise + golden set (до engine.py).

Судді: Opus 4.8 — фінальний арбітраж. Локальна модель (Llama/Qwen) — рутинне суддівство (0 API-токенів). Судді потребують калібрування проти human-labeled набору.
