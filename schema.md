# Канонічний шаблон агента · forge-sbs
*v1.0 · 2026-06-13 · синхронізовано KOVAL / MENTOR / CHEK*

> Єдине джерело правди для структури інструкцій.  
> KOVAL перевіряє наявність всіх 16 секцій перед видачею фінального блоку.  
> CHEK використовує цей шаблон як еталон при аудиті.  
> MENTOR зупиняється на ЕСКІЗІ (секції 1–6), фінальний артефакт — лише KOVAL.

## Пайплайн власності артефакту

```
MENTOR → ескіз (секції 1–4)
KOVAL  → повна інструкція (всі 16 секцій, фінальний блок)
CHEK   → аудит за цим шаблоном + першопринципи
```

MENTOR не видає фінальний блок. Якщо Серж просить «готовий текст» у Ментора — Ментор дає ескіз і передає до Коваля.

---

## Формат виводу інструкцій

**Plain text** — без markdown, без XML. Причина: всі три агенти обрали plain text, він більш крос-платформний.  
Markdown (✅/⚠️/❌, ```-фенси) — дозволений лише у *звітах* CHEKа, заборонений у *продукті* (готовій інструкції).

XML — лише для API-викликів engine (envelope-рівень), не для самого тексту інструкцій.

---

## Незалежність CHEK

CHEK перевіряє конформність шаблону (Haiku-рівень) + якість за **першопринципами**:
- чи агент може нашкодити без цього обмеження?
- чи правило виконується іншою моделлю без здогадок?
- чи немає надмірної автономності?

Реальна незалежність ≠ лише конформність шаблону.
